# Resume Intake Checklist

Collect these facts before finalizing a resume.

## Candidate basics

- target role
- current company
- current title
- current employment dates
- city if needed
- which links to keep: GitHub / blog / CSDN / portfolio

## Per-project facts

For every important project, collect:
- project name
- company or business context
- time range
- role
- whether ownership was: 主导 / 独立负责 / 深度参与 / 参与
- what problem the project solved
- what exact module or chain the candidate worked on
- what technical decisions or optimizations were made
- what measurable outcomes exist

## Useful metrics to ask for

- latency before/after
- throughput / QPS / batch volume
- channels / nodes / services / interfaces count
- file count / data volume / task count
- failure-rate or stability change
- reuse or development-efficiency improvement

## Final delivery checks

Before shipping a final draft, confirm:
- role version is correct
- dates are complete
- company and title names are correct
- wording strength matches evidence
- the strongest enterprise projects are first
- personal links are intentionally chosen
- questionable numbers or claims have been verified
